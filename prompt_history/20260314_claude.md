## GitHub MCP vs Web fetch — 比較結果

| 観点 | **GitHub MCP** | **Web fetch** |
|------|---------------|---------------|
| 速度 | ✅ 高速 | ❌ 429 (Rate Limit) でエラー |
| 日本語ファイル名 | ✅ そのまま渡せる（URLエンコード不要） | ⚠️ URLエンコード必須 |
| 出力 | ✅ **生のMarkdown/テキスト**がそのまま返る | HTML版（GitHubのUI込み）が返る |
| 長文ファイル | ⚠️ truncateされる場合あり（続きは読める） | ⚠️ HTML変換でノイズが多い |
| 認証 | ✅ PAT設定済みで自動 | ❌ publicでもRate Limitに引っかかりやすい |

**結論：GitHub MCP 一択です。**

理由：
1. Web fetchはGitHubのHTML版を取得するので、**Markdown本文以外のUI要素がノイズになる**。さらに今回429エラーで取れなかった
2. GitHub MCPは**生ファイルをそのまま返す**ので、マニュアルもプロンプト履歴もクリーンに読める
3. 日本語ファイル名（`課金プランとgoogleログインメモ.txt`）もそのまま渡せて楽
